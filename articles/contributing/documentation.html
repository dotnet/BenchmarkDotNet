Documentation

BenchmarkDotNet uses DocFX as a documentation generation tool.

Hints

  • If you want to provide a link to API, you can use cross references by UID. For example, [SimpleJobAttribute](xref:BenchmarkDotNet.Attributes.SimpleJobAttribute) and @BenchmarkDotNet.Attributes.SimpleJobAttribute will be transformed to SimpleJobAttribute.

Notes

DocFX uses the following syntax inside block quote for different types of notes:

> [!NOTE]
> note content
> [!TIP]
> tip content
> [!WARNING]
> warning content
> [!IMPORTANT]
> important content
> [!Caution]
> caution content

It will be transformed to:

Note

note content

Tip

tip content

Warning

warning content

Important

important content

Caution

caution content

Building documentation locally

You can build documentation locally with the help of the DocFX_Build Cake target. Use the DocFX_Serve Cake target to build and run the documentation.

Windows (PowerShell):

.\build.ps1 --target DocFX_Build
.\build.ps1 --target DocFX_Serve

Windows (Batch):

.\build.bat --target DocFX_Build
.\build.bat --target DocFX_Serve

Linux/macOS (Bash):

./build.sh --target DocFX_Build
./build.sh --target DocFX_Serve
